# 输入法管理功能

---

## 产品需求

### 1. 按窗口自动切换输入法

不同应用程序自动切换预设的输入法，不需要用户手动切。

**场景示例：**
- 切到微信 → 自动切微信输入法 + 中文
- 切到飞书 → 自动切微信输入法 + 英文
- 切到 VS Code → 自动切美式键盘

**行为要求：**
- 事件驱动（切窗口时触发），不轮询，不消耗 CPU
- 快速切换窗口时有防抖（300ms），不重复狂切
- 同一个程序的多个窗口间切换不触发（比如两个微信聊天窗口互切）
- 规则由用户自己设置，支持增删改
- ❌ 不需要自动恢复切走前的状态
- **自动切换时弹出 Toast 提示**，让用户知道已切到什么状态

### 2. 输入法状态变化实时提示

用户手动切换以下状态时，弹出 Toast 提示当前状态：

| 操作 | Toast 示例 |
|------|-----------|
| 按 CapsLock | 🔒 大写锁定 / 大写关闭 |
| Win+Space 切换输入法 | ⌨ 微信输入法 |
| 按 Shift 切换中英文 | 🇨🇳 中文 / 🇺🇸 英文 |
| 切换全角/半角 | 全角标点 / 半角标点 |

**行为要求：**
- Toast 简洁，不遮挡操作，自动消失
- 只提示当前状态，不需要自动恢复或纠正

### 3. 输入法状态管理设置页

在 WinAssistant 设置中新增「输入法状态管理」页签，包含：

**规则管理：**
- 列表展示已有规则（应用程序 + 输入法 + 语言 + 标点 + 大小写）
- 新增规则：输入进程名（或者从运行中的程序选取），选择输入法/语言/标点/大小写
- 删除规则（带确认）
- 支持编辑已有规则

**开关控制：**
- Toast 提示总开关
- 开启/关闭按窗口自动切换功能

**状态查看：**
- 一键显示当前输入法状态（当前窗口、当前输入法、中英文、全半角、大小写）

### 4. 非功能需求

- **零占用**：不切窗口时不消耗 CPU
- **低延迟**：窗口切换后 300ms 内完成切换
- **无外部依赖**：只用 Win32 API，不依赖第三方输入法 SDK
- **全局生效**：应用退出后不影响系统输入法设置
