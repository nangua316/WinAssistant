### 需要完成的改动

#### 1. Router.cs —— 始终级联

当前：只有 `matchedTrigger.Length <= 1` 时才级联其他 Agent。
改为：**无论触发词长度，只要主 Agent 返回 null，就级联到其他所有 Agent。**

原因：用户说"打开设置"时，Router 匹配 trigger"打开"→ AppAgent 找不到"设置"→ 因为没有级联直接返回 null。应该级联到 CompAgent，CompAgent 命中"设置"。

```csharp
// 改动 RouteAsync 方法中的第5步：
// 原代码只对短触发词级联
// 改为：只要 primary agent 返回 null，就尝试其他所有 agent（按 App→Comp→Browser→File 顺序）
// 但注意：不要级联到已经返回 null 的同一个 agent
```

**注意**：级联时 actionObject 不变，直接用当初提取的 actionObject 传给下一个 Agent。

#### 2. CompAgent.cs —— 添加热点/移动热点系统操作

```

```

#### 3. BrowserAgent.cs —— 实现默认浏览器检测

当前如果 actionObject 为空（用户只说"搜索"或"浏览器"），返回 null。
改为：如果 actionObject 为空，尝试查找默认浏览器并打开它。

实现 `FindDefaultBrowser()` 方法：
1. 尝试从注册表读取默认浏览器（用 C# 的 RegistryKey 或启动 edge.exe/chrome.exe）
2. 简单方案：硬编码回退到 `msedge.exe`，因为在 Windows 上一定有 Edge

```csharp
private static string? FindDefaultBrowser()
{
    // 先找 Edge（Windows 一定有）
    var edgePath = @"C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe";
    if (File.Exists(edgePath)) return edgePath;
    
    var altPath = @"C:\Program Files\Microsoft\Edge\Application\msedge.exe";
    if (File.Exists(altPath)) return altPath;
    
    return null;
}
```

并且在 TryMatchAsync 中，当 actionObject 为空时返回打开浏览器的结果。

#### 4. FileAgent.cs —— 更灵活的清理匹配

当前只匹配包含"垃圾"或"缓存"的 actionObject。
改为：
- 也匹配 "系统垃圾"、"临时文件"、"temp"、"系统清理"、"磁盘清理"、"C盘清理"
- 当匹配到清理相关请求时，返回 `cleanmgr /sagerun:1` 命令（用 `run_powershell` 类型）
- 当匹配到 "删除"、"找到"、"查找" 等但无具体文件名时，返回提示信息（用 `show_message` 类型告知用户暂未实现文件浏览功能）

#### 5. ChatWindow.xaml.cs —— Agent 执行结果展示优化

当前在 `SendMessage()` 方法中，Agent 执行后只显示 `routeResult.Reply` 和 `execResult.Message`。
改为：在 Agent 执行成功后，使用 `_library.Add(skill)` 将 Agent 匹配的技能保存到技能库，以便后续学习和复用。

改动位置：ChatWindow.xaml.cs 中 `SendMessage()` 方法的 `if (routeResult != null)` 分支，在 `_executor.ExecuteAsync` 成功后，调用 `_library.Add(skill)`。

注意：需要 **避免重复添加**——先检查是否已存在相同 ActionType + ActionParams 的技能。
